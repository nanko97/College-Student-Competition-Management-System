## 🎨 设计规范总结

### 颜色系统
```scss
$primary-color: #1890ff;      // 主色调
$primary-light: #e6f7ff;      // 浅色背景
$text-primary: #262626;       // 主要文字
$text-secondary: #595959;     // 次要文字
$text-muted: #8c8c8c;         // 弱化文字
$bg-color: #f0f2f5;           // 页面背景
```

### 圆角规范
```scss
$radius-sm: 4px;   // 小组件
$radius-md: 8px;   // 按钮、输入框
$radius-lg: 12px;  // 卡片
$radius-xl: 16px;  // 大卡片
```

### 动画曲线
```scss
$transition-fast: 0.15s cubic-bezier(0.4, 0, 0.2, 1);
$transition-normal: 0.3s cubic-bezier(0.4, 0, 0.2, 1);
$transition-slow: 0.5s cubic-bezier(0.4, 0, 0.2, 1);
```

### 阴影层级
```scss
$card-shadow: 0 2px 8px rgba(0, 0, 0, 0.09);        // 默认
$card-shadow-hover: 0 4px 16px rgba(0, 0, 0, 0.12); // 悬停
$card-shadow-lg: 0 8px 24px rgba(0, 0, 0, 0.15);    // 对话框
```

## 📝 技术要点

### 动画最佳实践
- ✅ 使用CSS3 transform而非top/left（GPU加速）
- ✅ 使用cubic-bezier缓动函数（更自然）
- ✅ 控制动画时长（快0.15s，中0.3s，慢0.5s）
- ✅ 避免同时执行多个复杂动画
- ✅ 移动端减少动画复杂度

### 性能优化
- ✅ 使用scoped样式避免污染
- ✅ 合理使用::v-deep覆盖Element UI样式
- ✅ 图片懒加载（后续实现）

## 💡 注意事项

1. **保持风格一致性**
   - 所有页面必须使用相同的颜色、圆角、阴影
   - 动画曲线统一使用cubic-bezier(0.4, 0, 0.2, 1)
   - 按钮、输入框、卡片样式保持一致

2. **后端接口优先**
   - 先查看后端Controller确定接口
   - 严格按照后端返回数据结构编写
   - 字段名必须完全匹配（区分大小写）

3. **测试驱动开发**
   - 每完成一个页面立即测试
   - 电脑端（1920×1080）测试
   - 移动端（iPhone 13）测试
   - 不同缩放比例测试（100%, 125%, 150%）

4. **渐进式优化**
   - 先保证功能正常
   - 再优化视觉效果
   - 最后添加动画细节
