# 竞赛管理业务联动优化方案

## 一、现状分析

### 1.1 当前问题
- ❌ 教师在"我的竞赛"中新增竞赛后，相关数据不联动
- ❌ 晋级关系、级别版本、赛道需要手动逐个创建
- ❌ 修改竞赛信息时，关联数据不会自动更新
- ❌ 各业务板块（赛道管理、晋级管理、级别认定）数据孤立

### 1.2 数据结构关系
```
jingsaixinxi (竞赛信息)
├── jingsai_saidao (赛道) - 通过 jingsaiId 关联
├── jingsai_jibie_banben (级别版本) - 通过 jingsaiId 关联
└── jingsai_jinji_guanxi (晋级关系) - 通过 fuJingsaiId/ziJingsaiId 关联
```

### 1.3 已有接口
- ✅ `JingsaixinxiController` - 竞赛CRUD
- ✅ `JingsaiSaidaoController` - 赛道CRUD
- ✅ `JingsaiJibieBanbenController` - 级别版本CRUD
- ✅ `JingsaiJinjiGuanxiController` - 晋级关系CRUD

## 二、优化目标

### 2.1 核心目标
**教师新增/修改竞赛时，自动联动创建/更新相关数据**

### 2.2 具体需求
1. **新增竞赛时**：
   - 教师填写竞赛基本信息（名称、类型、级别、时间等）
   - 自动创建默认赛道（如果"有赛道=是"）
   - 自动创建级别版本记录（如果级别为A/B/C类）
   - 自动初始化晋级关系（如果"需晋级=是"）

2. **修改竞赛时**：
   - 修改竞赛基本信息
   - 联动更新赛道信息（名称、类型等）
   - 联动更新级别版本
   - 联动更新晋级关系

3. **删除竞赛时**：
   - 级联删除相关赛道、级别版本、晋级关系

## 三、实现方案

### 3.1 方案一：后端自动联动（推荐）

#### 3.1.1 修改 `JingsaixinxiController.save()` 方法
```java
@RequestMapping("/save")
@Transactional(rollbackFor = Exception.class)
public R save(@RequestBody JingsaixinxiEntity jingsaixinxi, HttpServletRequest request) {
    // 1. 保存竞赛基本信息（原有逻辑）
    jingsaixinxi.setId(IdWorker.getId());
    boolean result = jingsaixinxiService.insert(jingsaixinxi);
    
    if (!result) {
        return R.error("保存失败");
    }
    
    Long jingsaiId = jingsaixinxi.getId();
    
    // 2. 如果"有赛道=是"，自动创建默认赛道
    if ("是".equals(jingsaixinxi.getShifouYouSaidao())) {
        createDefaultSaidao(jingsaiId, jingsaixinxi);
    }
    
    // 3. 自动创建级别版本记录
    if (StringUtils.hasText(jingsaixinxi.getJingsaiJibie())) {
        createJibieBanben(jingsaiId, jingsaixinxi);
    }
    
    // 4. 如果"需晋级=是"，创建晋级关系占位符（后续由教师完善）
    if ("是".equals(jingsaixinxi.getShifouXuyaoJinji())) {
        createJinjiGuanxiPlaceholder(jingsaiId, jingsaixinxi);
    }
    
    return R.ok("保存成功，已自动创建相关数据");
}

/**
 * 创建默认赛道
 */
private void createDefaultSaidao(Long jingsaiId, JingsaixinxiEntity jingsaixinxi) {
    JingsaiSaidaoEntity saidao = new JingsaiSaidaoEntity();
    saidao.setId(IdWorker.getId());
    saidao.setJingsaiId(jingsaiId);
    saidao.setJingsaimingcheng(jingsaixinxi.getJingsaimingcheng());
    saidao.setSaidaoMingcheng("默认赛道");
    saidao.setSaidaoBianma("SD001");
    saidao.setCansaiLeixing("个人赛");
    saidao.setIsActive("是");
    saidao.setSortOrder(1);
    saidao.setAddtime(new Date());
    
    saidaoService.insert(saidao);
    log.info("自动创建默认赛道成功，竞赛ID: {}, 赛道ID: {}", jingsaiId, saidao.getId());
}

/**
 * 创建级别版本记录
 */
private void createJibieBanben(Long jingsaiId, JingsaixinxiEntity jingsaixinxi) {
    JingsaiJibieBanbenEntity jibie = new JingsaiJibieBanbenEntity();
    jibie.setId(IdWorker.getId());
    jibie.setJingsaiId(jingsaiId);
    jibie.setJingsaimingcheng(jingsaixinxi.getJingsaimingcheng());
    jibie.setJingsaiNianfen(jingsaixinxi.getNianfen());
    jibie.setJibie(jingsaixinxi.getJingsaiJibie());
    jibie.setIsCurrent("是");
    jibie.setAddtime(new Date());
    
    jibieBanbenService.insert(jibie);
    log.info("自动创建级别版本成功，竞赛ID: {}, 级别版本ID: {}", jingsaiId, jibie.getId());
}

/**
 * 创建晋级关系占位符（需要教师后续完善）
 */
private void createJinjiGuanxiPlaceholder(Long jingsaiId, JingsaixinxiEntity jingsaixinxi) {
    // 这里只创建占位符，具体晋级规则由教师在"晋级管理"中完善
    log.info("竞赛需晋级，请在晋级管理中配置晋级规则，竞赛ID: {}", jingsaiId);
}
```

#### 3.1.2 修改 `JingsaixinxiController.update()` 方法
```java
@RequestMapping("/update")
@Transactional(rollbackFor = Exception.class)
public R update(@RequestBody JingsaixinxiEntity jingsaixinxi, HttpServletRequest request) {
    // 1. 查询原竞赛信息
    JingsaixinxiEntity oldJingsai = jingsaixinxiService.selectById(jingsaixinxi.getId());
    
    // 2. 更新竞赛基本信息（原有逻辑）
    jingsaixinxiService.updateById(jingsaixinxi);
    
    Long jingsaiId = jingsaixinxi.getId();
    
    // 3. 联动更新赛道
    if ("是".equals(jingsaixinxi.getShifouYouSaidao())) {
        updateSaidaoByJingsai(jingsaiId, jingsaixinxi);
    } else {
        // 如果改为"无赛道"，删除相关赛道
        deleteSaidaoByJingsai(jingsaiId);
    }
    
    // 4. 联动更新级别版本
    updateJibieBanbenByJingsai(jingsaiId, jingsaixinxi);
    
    // 5. 联动更新晋级关系
    if ("是".equals(jingsaixinxi.getShifouXuyaoJinji())) {
        updateJinjiGuanxiByJingsai(jingsaiId, jingsaixinxi);
    } else {
        // 如果改为"不需晋级"，删除相关晋级关系
        deleteJinjiGuanxiByJingsai(jingsaiId);
    }
    
    return R.ok("修改成功，已联动更新相关数据");
}
```

#### 3.1.3 修改 `JingsaixinxiController.delete()` 方法
```java
@RequestMapping("/delete")
@Transactional(rollbackFor = Exception.class)
public R delete(@RequestBody Long[] ids) {
    for (Long jingsaiId : ids) {
        // 1. 删除相关赛道
        deleteSaidaoByJingsai(jingsaiId);
        
        // 2. 删除相关级别版本
        deleteJibieBanbenByJingsai(jingsaiId);
        
        // 3. 删除相关晋级关系
        deleteJinjiGuanxiByJingsai(jingsaiId);
        
        // 4. 删除竞赛本身
        jingsaixinxiService.deleteById(jingsaiId);
    }
    
    return R.ok("删除成功，已级联删除相关数据");
}
```

### 3.2 方案二：前端引导创建（备选）

在竞赛新增表单中增加"关联数据配置"步骤：
1. 步骤1：填写竞赛基本信息
2. 步骤2：配置赛道信息（可添加多个）
3. 步骤3：配置级别版本
4. 步骤4：配置晋级关系

**优点**：教师可自定义配置
**缺点**：操作流程复杂，用户体验差

### 3.3 推荐方案：方案一 + 灵活配置

- 默认自动创建基础数据（赛道、级别版本）
- 教师可在各业务板块继续完善和修改
- 提供"重新生成关联数据"按钮

## 五、数据库注意事项

### 5.1 外键约束
当前数据库**没有设置外键约束**，需要：
- 在代码层面保证数据一致性
- 删除竞赛时手动删除关联数据

### 5.2 事务处理
所有联动操作必须在同一个事务中：
```java
@Transactional(rollbackFor = Exception.class)
public R save(...) {
    // 所有数据库操作
}
```

### 5.3 性能优化
- 批量操作使用 `insertBatch`
- 避免 N+1 查询问题
- 添加数据库索引（jingsaiId 字段）

## 六、风险评估

### 6.1 数据一致性风险
- **风险**：联动创建失败导致数据不一致
- **解决**：使用事务回滚，确保原子性

### 6.2 性能风险
- **风险**：大量关联数据导致性能下降
- **解决**：异步处理非关键操作，添加日志监控

### 6.3 用户体验风险
- **风险**：自动创建的数据不符合教师预期
- **解决**：创建后提供"编辑"入口，允许教师修改

## 七、验收标准

### 7.1 功能验收
- [ ] 教师新增竞赛后，自动创建默认赛道
- [ ] 教师新增竞赛后，自动创建级别版本记录
- [ ] 教师修改竞赛名称，相关赛道名称同步更新
- [ ] 教师删除竞赛，相关数据全部删除
- [ ] 教师只能看到自己创建的竞赛的关联数据

### 7.2 性能验收
- [ ] 新增竞赛接口响应时间 < 2秒
- [ ] 批量删除10个竞赛响应时间 < 5秒
- [ ] 数据库事务无死锁

### 7.3 数据验收
- [ ] 无孤立数据（竞赛删除后，关联数据全部删除）
- [ ] 无重复数据（多次修改不产生重复记录）
- [ ] 数据一致性（竞赛名称修改后，所有关联表同步更新）
