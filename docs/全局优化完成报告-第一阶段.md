```java
public class ConfigController {
    
    @OperationLog("保存系统配置")
    @PostMapping("/save")
    @Transactional(rollbackFor = Exception.class)
    public R save(@RequestBody ConfigEntity config) {
        try {
            if (!StringUtils.hasText(config.getName())) {
                return R.error("配置名称不能为空");
            }
            configService.insert(config);
            log.info("保存配置成功，名称: {}", config.getName());
            return R.ok("保存成功");
        } catch (Exception e) {
            log.error("保存配置异常", e);
            return R.error("保存失败，请重试");
        }
    }
}
```

## 💡 优化最佳实践总结

通过本次优化，我们确立了以下最佳实践标准：

### 1. 事务管理规范
```java
@Transactional(rollbackFor = Exception.class)
public R update(...) { ... }
```
- 所有修改操作必须添加事务
- 指定 rollbackFor = Exception.class
- 确保多表操作的原子性

### 2. 参数校验规范
```java
if (entity.getId() == null || entity.getId() <= 0) {
    return R.error("ID非法");
}
if (!StringUtils.hasText(entity.getName())) {
    return R.error("名称不能为空");
}
```
- ID有效性校验
- 必填字段非空校验
- 业务规则校验

### 3. 异常处理规范
```java
try {
    // 业务逻辑
} catch (Exception e) {
    log.error("操作失败", e);
    return R.error("操作失败，请重试");
}
```
- 所有方法必须有try-catch
- 记录完整异常堆栈
- 返回友好错误提示

### 4. 日志记录规范
```java
log.info("操作成功，ID: {}, 名称: {}", id, name);
log.warn("警告信息：{}", detail);
log.error("操作失败", e);
```
- 入口记录关键参数
- 成功记录结果
- 失败记录异常

### 5. 操作日志规范
```java
@OperationLog("操作描述")
public R xxx(...) { ... }
```
- 所有修改操作添加操作日志
- 便于审计追踪
