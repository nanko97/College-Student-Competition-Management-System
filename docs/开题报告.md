## 一、研究背景与意义

### 1.1 研究背景

随着高等教育改革的不断深化，大学生学科竞赛已成为培养创新人才的重要载体。当前，高校竞赛种类繁多，包括学科竞赛、创新创业大赛、科技作品竞赛、艺术设计比赛、体育竞技等多个类别。传统的竞赛管理模式主要依靠人工操作和纸质文档，存在以下问题：

1. **信息管理分散**: 竞赛通知通过不同渠道发布，学生难以全面获取信息
2. **报名流程繁琐**: 学生需要填写纸质报名表，教师手动审核，效率低下
3. **数据统计困难**: 参赛人数、获奖情况等数据难以实时统计和分析
4. **过程监管缺失**: 无法有效跟踪竞赛各环节的进展情况
5. **资源共享不足**: 优秀作品和获奖经验难以传承和分享

在"互联网 + 教育"的背景下，利用信息化手段改进竞赛管理已成为高校教学改革的必然趋势。基于 Web 的竞赛管理系统能够实现竞赛信息的集中发布、在线报名、作品提交、评分管理等功能，提高管理效率，优化参赛体验。

### 1.2 研究意义

#### 1.2.1 理论意义

本系统采用 Spring Boot 和 Vue.js 框架进行开发，实践了前后端分离的软件工程思想，体现了现代 Web 应用开发的先进理念。系统中融入了 RBAC(基于角色的访问控制)权限模型、AOP(面向切面编程)日志记录、XSS 攻击防护等安全技术，为高校信息系统的安全设计提供了参考案例。

#### 1.2.2 实际意义

1. **提升管理效率**: 实现竞赛全流程电子化管理，减少人工操作，提高工作效率
2. **优化参赛体验**: 学生可在线浏览竞赛信息、一键报名、查看成绩，操作便捷
3. **促进信息公开**: 所有竞赛信息统一发布，确保信息透明，避免遗漏
4. **支持决策分析**: 提供数据统计功能，为教学改革和资源分配提供依据
5. **保障系统安全**: 实施多重安全防护措施，保护用户数据和隐私

### 1.3 国内外研究现状

#### 1.3.1 国外研究现状

国外高校在竞赛管理方面起步较早，如美国麻省理工学院(MIT)开发了完整的创新项目管理系统，支持项目申请、团队组建、进度跟踪等功能。欧洲多所大学采用开源平台如 Devpost 举办黑客松活动，实现了在线报名、作品展示和评审的一体化管理。这些系统普遍具有以下特点:

- 高度集成化，与学校教务系统无缝对接
- 注重用户体验，界面友好，操作简便
- 强调协作功能，支持跨校、跨国团队合作
- 数据分析能力强，提供可视化报表

#### 1.3.2 国内研究现状

国内高校竞赛管理系统建设相对滞后，但近年来发展迅速。部分"双一流"高校已自主开发了竞赛管理平台，如清华大学的"科创管理系统"、浙江大学的"竞赛管理信息平台"等。然而，大多数地方院校仍采用传统管理方式或简单的 Excel 表格管理，存在信息化程度低、功能单一等问题。

现有系统的主要不足:
- 功能局限于信息发布，缺乏全流程管理
- 安全性考虑不足，存在数据泄露风险
- 移动端适配差，无法满足随时随地的使用需求
- 扩展性不强，难以适应竞赛类型的变化

#### 1.3.3 发展趋势

未来高校竞赛管理系统将呈现以下发展趋势:

1. **智能化**: 引入 AI 技术实现智能推荐、自动评分等功能
2. **移动化**: 开发移动应用，支持手机、平板等多终端访问
3. **云化**: 采用云计算架构，实现弹性扩展和高可用
4. **生态化**: 构建竞赛生态系统，整合培训、辅导、就业等资源

### 3.4 安全设计

#### 3.4.1 身份认证机制

采用 Token 认证机制，流程如下:

1. 用户登录成功后，服务端生成包含用户 ID、角色、过期时间的 Token
2. Token 存储到数据库 token 表，有效期 1 小时
3. 客户端将 Token 存储在 localStorage，后续请求自动携带
4. 服务端拦截器验证 Token 有效性，无效则返回 401

**Token 结构**:
```java
public class TokenEntity {
    private Long userid;        // 用户 ID
    private String username;    // 用户名
    private String tablename;   // 表名 (角色标识)
    private String role;        // 角色名称
    private String token;       // JWT 风格随机字符串
    private Date expiretime;    // 过期时间
}
```

#### 3.4.2 权限控制设计

基于 RBAC 模型的权限控制:

1. **角色定义**: 学生 (xuesheng)、教师 (jiaoshi)、管理员 (admin)
2. **权限表**: user_role_permission 表存储角色与资源的映射关系
3. **权限验证**: 拦截器根据用户角色和请求 URL 验证权限

**权限配置示例**:
```java
@Component
public class AuthorizationInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) {
        // 1. 检查是否需要忽略权限
        IgnoreAuth annotation = ((HandlerMethod) handler)
            .getMethodAnnotation(IgnoreAuth.class);
        if (annotation != null) {
            return true;
        }
        
        // 2. 验证 Token
        String token = request.getHeader("Token");
        TokenEntity tokenEntity = tokenService.getTokenEntity(token);
        
        if (tokenEntity == null) {
            response.setStatus(401);
            return false;
        }
        
        // 3. 验证权限
        if (!userPermissionService.hasPermission(
                tokenEntity.getRole(), 
                request.getRequestURI())) {
            response.setStatus(403);
            return false;
        }
        
        return true;
    }
}
```

#### 3.4.3 XSS 攻击防护

实现 XSS 过滤器，自动过滤用户输入中的恶意脚本:

```java
@Component
public class XssFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                        FilterChain chain) throws IOException, ServletException {
        
        if (request instanceof HttpServletRequest) {
            HttpServletRequest httpRequest = (HttpServletRequest) request;
            
            // 使用 XSS 包装器包装请求，自动过滤危险内容
            XssHttpServletRequestWrapper xssRequest = 
                new XssHttpServletRequestWrapper(httpRequest);
            
            chain.doFilter(xssRequest, response);
        } else {
            chain.doFilter(request, response);
        }
    }
}

// HTML 标签过滤
private String cleanXSS(String value) {
    return value.replaceAll("<", "&lt;")
                .replaceAll(">", "&gt;")
                .replaceAll("\\(", "&#40;")
                .replaceAll("'", "&#39;")
                .replaceAll("eval\\((.*)\\)", "")
                .replaceAll("[\\\"\\\'][\\s]*javascript:(.*)[\\\"\\\']", "\"\"\"");
}
```

#### 3.4.4 密码加密存储

采用 BCrypt 算法对用户密码进行加密存储:

```java
@Service
public class UserServiceImpl implements UserService {
    
    private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();
    
    @Override
    @Transactional
    public void register(UserEntity user) {
        // 密码加密
        user.setPassword(passwordEncoder.encode(user.getPassword()));
        userDao.insert(user);
    }
    
    @Override
    public UserEntity login(String username, String password) {
        UserEntity user = userDao.selectOne(username);
        
        // 密码验证
        if (!passwordEncoder.matches(password, user.getPassword())) {
            throw new AuthenticationException("密码错误");
        }
        
        return user;
    }
}
```

#### 3.4.5 操作日志记录

基于 AOP 实现操作日志自动记录:

```java
@Aspect
@Component
@Slf4j
public class OperationLogAspect {
    
    @Around("@annotation(com.annotation.OperationLog)")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        long startTime = System.currentTimeMillis();
        
        // 获取操作注解信息
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        Method method = signature.getMethod();
        OperationLog operationLog = method.getAnnotation(OperationLog.class);
        
        // 获取请求信息
        HttpServletRequest request = ((ServletRequestAttributes) 
            RequestContextHolder.getRequestAttributes()).getRequest();
        
        String operator = getOperator(request);
        String methodName = method.getName();
        String description = operationLog.value();
        
        // 打印操作开始日志
        log.info("\n===== 操作开始 =====");
        log.info("操作人：{}", operator);
        log.info("操作方法：{}", methodName);
        log.info("操作描述：{}", description);
        
        try {
            Object result = joinPoint.proceed();
            
            long endTime = System.currentTimeMillis();
            log.info("===== 操作成功 =====");
            log.info("耗时：{}ms", endTime - startTime);
            
            return result;
        } catch (Throwable e) {
            log.error("===== 操作失败 =====", e);
            throw e;
        }
    }
}
```

### 4.1 数据库概念设计

系统采用 MySQL 5.7+ 数据库，主要包含以下实体:

1. **用户 (User)**: 系统登录用户，包含用户名、密码、角色
2. **学生 (Xuesheng)**: 学生详细信息，包含学号、姓名、班级等
3. **教师 (Jiaoshi)**: 教师详细信息，包含工号、姓名、职称等
4. **竞赛信息 (Jingsaixinxi)**: 竞赛基本信息，包含名称、类型、时间等
5. **竞赛报名 (Jingsaibaoming)**: 学生报名信息，包含参赛人员、审核状态等
6. **作品打分 (Zuopindafen)**: 作品评分信息，包含分数、评语等
7. **Token 令牌 (Token)**: 用户登录凭证，包含 Token、用户 ID、过期时间
8. **角色权限 (UserRolePermission)**: 角色与权限的映射关系

## 五、系统特色与创新点

### 5.1 系统特色

#### 5.1.1 工程规范性

1. **分层架构**: 严格遵循 Controller-Service-DAO 三层架构，职责清晰
2. **统一规范**: 统一返回格式、统一异常处理、统一命名规范
3. **代码注释**: 详细的中文注释，类、方法、字段注释覆盖率超过 80%
4. **文档齐全**: 提供完整的需求文档、设计文档、部署文档

#### 5.1.2 安全可靠性

1. **多重认证**: Token 认证 + 角色权限验证 + 操作日志审计
2. **数据加密**: BCrypt 密码加密 + XSS 攻击防护 + SQL 注入防护
3. **防暴力破解**: 登录失败次数限制，防止恶意攻击
4. **异常处理**: 全局异常捕获，避免敏感信息泄露

#### 5.1.3 用户体验

1. **实时反馈**: 密码强度实时检测、表单验证即时提示
2. **友好界面**: Element UI 组件库，界面简洁美观
3. **操作便捷**: 一键报名、批量操作、快捷搜索
4. **响应式设计**: 适配 PC、平板、手机等多种设备

#### 5.1.4 可维护性

1. **模块化设计**: 功能模块独立，便于维护和扩展
2. **AOP 日志**: 操作日志自动记录，便于问题排查
3. **配置外置**: 配置文件与代码分离，支持多环境部署
4. **数据库版本管理**: Flyway 迁移脚本，数据库变更可追溯

### 5.2 创新点

#### 5.2.1 密码强度实时检测

传统系统通常在提交后才验证密码强度，本系统在注册页面实时检测密码强度并可视化展示:

**技术创新**:
- 前端 JavaScript 算法实时计算密码强度分数
- 进度条 + 颜色 + 文字多维度展示强度等级
- 详细列出字符类型要求 (大写/小写/数字/特殊字符)

**应用价值**:
- 提升用户密码安全意识
- 减少因密码不符合要求导致的重复提交
- 增强用户体验

#### 5.2.2 XSS 攻击防护过滤器

针对 Web 应用常见的 XSS 攻击，实现自定义过滤器:

**技术创新**:
- 基于 Servlet Filter 实现请求拦截
- 使用 HttpServletRequestWrapper 包装请求
- 自动过滤 HTML 标签、JavaScript 代码、事件处理器

**应用价值**:
- 防止恶意脚本注入
- 保护用户数据安全
- 符合 OWASP Top 10 防护要求

#### 5.2.3 基于 AOP 的操作日志

采用面向切面编程思想，实现操作日志自动记录:

**技术创新**:
- 自定义@OperationLog 注解标记需要记录的操作
- Around 环绕通知拦截方法执行
- 自动记录操作人、操作时间、操作内容、请求参数、执行耗时

**应用价值**:
- 无需修改业务代码即可添加日志功能
- 便于问题排查和责任追溯
- 展示 AOP 编程思想的应用

#### 5.2.4 多角色权限控制

基于 RBAC 模型实现灵活的权限管理:

**技术创新**:
- 角色与权限分离设计，支持动态配置
- 拦截器统一验证权限，减少代码重复
- 支持@IgnoreAuth 注解忽略特定接口权限

**应用价值**:
- 满足不同角色的差异化需求
- 提升系统安全性
- 便于权限扩展和调整
