## 1. 项目概述

### 1.1 系统目标

本系统旨在为高校提供一个完整的竞赛管理平台，实现竞赛信息发布、学生报名、作品提交、教师评分等全流程管理。系统采用 B/S 架构，支持多角色访问，具备良好的可扩展性和可维护性。

### 1.2 适用场景

- **高校教务处**: 管理各类学科竞赛活动
- **参赛学生**: 浏览竞赛信息、在线报名、查看成绩
- **指导教师**: 发布竞赛、审核报名、评审作品
- **系统管理员**: 用户管理、权限分配、系统配置

### 1.3 核心功能模块

| 模块名称 | 功能描述 | 主要用户 |
|----------|----------|----------|
| 用户管理 | 注册、登录、个人信息管理 | 所有用户 |
| 竞赛信息管理 | 竞赛发布、编辑、查询、删除 | 教师、管理员 |
| 竞赛报名管理 | 在线报名、报名审核、状态查询 | 学生、教师 |
| 作品打分管理 | 作品提交、评分、评语填写 | 教师 |
| 系统配置 | 参数设置、班级管理 | 管理员 |

### 4.4 权限验证流程

```
1. 客户端请求 → 2. 拦截器捕获 → 3. 检查@IgnoreAuth 注解
                                          ↓ (无注解)
4. 从 Header 获取 Token → 5. TokenService 验证有效性
                                          ↓ (有效)
6. 从 Session 获取角色 → 7. UserPermissionService 验证权限
                                          ↓
8. 有权限 → 放行    9. 无权限 → 返回 403
```

**拦截器实现**:
```java
@Component
public class AuthorizationInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) {
        // 1. 检查是否需要忽略权限
        IgnoreAuth annotation = ((HandlerMethod) handler)
            .getMethodAnnotation(IgnoreAuth.class);
        if (annotation != null) {
            return true;
        }
        
        // 2. 验证 Token
        String token = request.getHeader("Token");
        TokenEntity tokenEntity = tokenService.getTokenEntity(token);
        
        if (tokenEntity == null) {
            response.setStatus(401);
            return false;
        }
        
        // 3. 验证权限
        if (!userPermissionService.hasPermission(
                tokenEntity.getRole(), 
                request.getRequestURI())) {
            response.setStatus(403);
            return false;
        }
        
        return true;
    }
}
```

## 5. 安全设计

### 5.1 登录认证机制

#### Token 认证流程

```
1. 用户输入账号密码 → 2. 后端验证 → 3. 生成 Token
                                              ↓
4. 客户端存储 Token → 5. 后续请求携带 Token → 6. 服务端验证
```

#### Token 结构

| 字段 | 类型 | 描述 |
|------|------|------|
| `token` | String | JWT 风格随机字符串 |
| `userid` | Long | 用户 ID |
| `username` | String | 用户名 |
| `tablename` | String | 表名 (角色标识) |
| `role` | String | 角色名称 |
| `expiretime` | Date | 过期时间 (1 小时) |

**Token 生成代码**:
```java
public String generateToken(Long userId, String username, 
                           String tableName, String role) {
    TokenEntity tokenEntity = new TokenEntity();
    tokenEntity.setUserid(userId);
    tokenEntity.setUsername(username);
    tokenEntity.setTablename(tableName);
    tokenEntity.setRole(role);
    tokenEntity.setToken(UUID.randomUUID().toString().replace("-", ""));
    tokenEntity.setExpiretime(new Date(System.currentTimeMillis() + 3600000));
    
    tokenService.createToken(tokenEntity);
    return tokenEntity.getToken();
}
```

### 5.2 密码加密 (BCrypt)

#### 加密策略

- **算法**: BCrypt (Spring Security Crypto)
- **强度**: 10 轮迭代
- **特点**: 加盐哈希、不可逆、抗彩虹表攻击

**加密代码**:
```java
private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

// 注册时加密
user.setPassword(passwordEncoder.encode(user.getPassword()));

// 登录时验证
if (!passwordEncoder.matches(password, user.getPassword())) {
    return R.error("账号或密码不正确");
}
```

### 5.5 登录安全控制

#### 防暴力破解

```yaml

  max-fail-count: 5  # 失败次数限制
```

```java
// 检查登录失败次数
Integer failCount = loginFailCountMap.getOrDefault(username, 0);

}

// 登录成功后重置计数
loginFailCountMap.remove(username);
```

**密码强度增强版**:
```java
public class PasswordValidator {
    
    private static final int MIN_LENGTH = 8;
    private static final Pattern UPPER_CASE = Pattern.compile("[A-Z]");
    private static final Pattern LOWER_CASE = Pattern.compile("[a-z]");
    private static final Pattern DIGIT = Pattern.compile("\\d");
    private static final Pattern SPECIAL_CHAR = Pattern.compile("[^A-Za-z0-9]");
    
    public static boolean isValid(String password) {
        if (!StringUtils.hasText(password) || password.length() < MIN_LENGTH) {
            return false;
        }
        
        // 至少包含大写字母、小写字母、数字、特殊字符中的三种
        int matchCount = 0;
        if (UPPER_CASE.matcher(password).find()) matchCount++;
        if (LOWER_CASE.matcher(password).find()) matchCount++;
        if (DIGIT.matcher(password).find()) matchCount++;
        if (SPECIAL_CHAR.matcher(password).find()) matchCount++;
        
        return matchCount >= 3;
    }
    
    public static String getRequirements() {
        return "密码长度不少于 8 位，且包含大写字母、小写字母、数字、特殊字符中的至少三种";
    }
}
```

### C. 常见问题 FAQ

**Q1: 数据库连接失败？**  
A: 检查 MySQL 服务是否启动、数据库是否存在、用户名密码是否正确。

**Q2: 端口被占用？**  
A: 修改 `application.yml` 中的 `server.port` 为其他端口 (如 8081)。

**Q3: 前端页面无法加载？**  
A: 确认前端已构建 (`npm run build`)，检查 `target/classes/admin/admin/dist` 目录是否存在。

**Q4: 登录提示"请先登录"？**  
A: 检查请求 Header 是否携带 Token，Token 是否过期 (有效期 1 小时)。

**Q5: 密码无法登录？**  
A: 确认密码已使用 BCrypt 加密存储，旧数据需重置密码。
