## 五、响应式断点

### 5.1 断点定义

```css
/* 桌面端 */
@media screen and (min-width: 1201px) {
  /* 完整布局 */
}

/* 平板端 */
@media screen and (max-width: 1200px) {
  .form-content .el-row.slt .el-form-item {
    width: 48%; /* 两列布局 */
  }
}

/* 手机端横屏 */
@media screen and (max-width: 768px) {
  .form-content .el-row.slt .el-form-item {
    width: 100%; /* 单列布局 */
  }
}

/* 手机端竖屏 */
@media screen and (max-width: 480px) {
  .el-button {
    width: 100%; /* 按钮全宽 */
  }
}
```

### 5.2 适配策略

| 屏幕宽度 | 布局策略 | 字体调整 |
|----------|----------|----------|
| > 1200px | 多列布局 | 标准 14px |
| 768-1200px | 两列布局 | 标准 14px |
| 480-768px | 单列布局 | 缩小到 13px |
| < 480px | 单列布局 + 全宽按钮 | 缩小到 12px |

## 六、性能优化

### 6.1 CSS 优化

1. **使用 CSS 变量** - 便于主题切换
2. **避免深度嵌套** - 最多 3 层嵌套
3. **使用简写属性** - 减少代码量
4. **合理使用 inherit** - 继承父元素样式

### 6.2 动画优化

1. **使用 transform** - GPU 加速，性能更好
2. **避免 animate 大量元素** - 只动画关键元素
3. **will-change 提示** - 提前告知浏览器优化

```css
.el-button--primary {
  will-change: transform; /* 提示浏览器优化 */
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

### 6.3 加载优化

1. **按需加载** - 只加载需要的样式
2. **关键 CSS 内联** - 首屏样式内联
3. **非关键 CSS 异步** - 异步加载装饰性样式

## 十一、常见问题

### Q1: 样式没有生效？

**A**: 清除浏览器缓存（Ctrl+Shift+Delete），然后刷新页面。

### Q2: 动画卡顿？

**A**: 检查电脑性能，或临时禁用动画：

```css
* {
  animation: none !important;
  transition: none !important;
}
```

### Q3: 移动端样式异常？

**A**: 确保 meta viewport 设置正确：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### Q4: 打印时样式混乱？

**A**: 已内置打印样式，使用 Ctrl+P 预览效果。
