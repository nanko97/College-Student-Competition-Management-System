## 二、发现的问题

### ⚠️ 问题 1：静态资源路径配置风险

**位置**: `InterceptorConfig.java` 第 52-60 行

**问题描述**:
```java
@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
    registry.addResourceHandler("/**")
            .addResourceLocations("classpath:/resources/")
            .addResourceLocations("classpath:/static/")
            .addResourceLocations("classpath:/admin/")  // ⚠️ 可能不存在
            .addResourceLocations("classpath:/front/")  // ⚠️ 可能不存在
            .addResourceLocations("classpath:/public/");
    super.addResourceHandlers(registry);
}
```

**风险**:
- `classpath:/admin/` 和 `classpath:/front/` 目录可能不存在
- 可能导致 404 错误
- 影响前端页面访问

**检查方法**:
```bash
ls src/main/resources/admin
ls src/main/resources/front
```

**修复方案**:
```java
@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
    registry.addResourceHandler("/**")
            .addResourceLocations("classpath:/resources/")
            .addResourceLocations("classpath:/static/")
            // 只添加实际存在的目录
            .addResourceLocations("file:src/main/resources/admin/")
            .addResourceLocations("classpath:/public/");
    super.addResourceHandlers(registry);
}
```

**当前状态**: ⚠️ **需要修复**

## 九、常见问题

### Q1: 启动后端时报"端口被占用"？

**A**: 修改 application.yml 中的端口：
```yaml
server:
  port: 8081  # 改为其他端口
```

### Q2: 前端启动失败？

**A**: 
1. 检查 Node.js 版本（建议 14+）
2. 删除 node_modules 重新安装：
```bash
rm -rf node_modules
npm install
```

### Q3: 登录页面样式异常？

**A**: 清除浏览器缓存或重启浏览器

### Q4: 文件上传失败？

**A**: 检查 upload 目录是否存在，是否有写权限

### Q5: 数据库连接失败？

**A**: 
1. 检查 MySQL 服务是否启动
2. 检查数据库用户名密码是否正确
3. 检查数据库是否存在
