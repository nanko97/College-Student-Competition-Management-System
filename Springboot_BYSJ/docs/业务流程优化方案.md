## 👥 二、角色功能详解

### 2.1 教师角色（Jiaoshi）

#### 核心职责
1. **发布竞赛** - 创建新的竞赛项目
2. **审核报名** - 审批学生的报名申请
3. **查看报名情况** - 监控所发布竞赛的报名状态
4. **作品评分** - 对参赛作品进行打分

#### 接口权限

| 报名管理 | `/jingsaibaoming/*` | 审核、查看、删除 | `JingsaibaomingController.java` |
| 作品评分 | `/zuopindafen/*` | 评分、查看、修改 | `ZuopindafenController.java` |
| 学生查看 | `/xuesheng/list` | 查看学生列表 | `XueshengController.java` |

#### 操作流程

**发布竞赛流程：**
```java
// JingsaixinxiController.java:120
@RequestMapping("/save")
public R save(@RequestBody JingsaixinxiEntity jingsaixinxi, HttpServletRequest request) {
    // 1. 自动填充教师工号
    String gonghao = (String) request.getSession().getAttribute("username");
    jingsaixinxi.setGonghao(gonghao);
    
    // 2. 验证竞赛信息
    ValidatorUtils.validateEntity(jingsaixinxi);
    
    // 3. 生成 ID 并发布
    jingsaixinxi.setId(IdWorker.getId());
    jingsaixinxiService.insert(jingsaixinxi);
    
    return R.ok("竞赛发布成功");
}
```

**审核报名流程：**
```java
// JingsaibaomingController.java:470
@RequestMapping("/shenhue")
public R shenhue(@RequestBody Map<String, Object> params) {
    Long id = Long.parseLong(params.get("id").toString());
    String sfsh = params.get("sfsh").toString();
    String shhf = params.get("shhf").toString();
    
    JingsaibaomingEntity jingsaibaoming = jingsaibaomingService.selectById(id);
    jingsaibaoming.setSfsh(sfsh);
    jingsaibaoming.setShhf(shhf);
    
    jingsaibaomingService.updateById(jingsaibaoming);
    return R.ok("审核成功");
}
```

### 4.3 事务处理

```java
// UserRegistrationServiceImpl.java:39-65
@Override
@Transactional(rollbackFor = Exception.class)
public RegisterResultVO register(UserRegisterDTO dto) {
    // 1. 验证账号唯一性
    if (!validateAccountUnique(dto.getRole(), dto.getAccount())) {
        return RegisterResultVO.error("账号已存在");
    }
    
    // 2. 根据角色创建对应实体
    Long userId = IdWorker.getId();
    String role = dto.getRole();
    
    // 3. 统一角色名称
    if ("xuesheng".equals(role)) role = "学生";
    else if ("jiaoshi".equals(role)) role = "教师";
    else if ("guanliyuan".equals(role)) role = "管理员";
    
    // 4. 创建用户记录（同时写入 users 表）
    switch (role) {
        case "学生": return registerStudent(dto, userId);
        case "教师": return registerTeacher(dto, userId);
        case "管理员": return registerAdmin(dto, userId);
        default: return RegisterResultVO.error("不支持的角色类型");
    }
}
```

## 📞 八、技术支持

### 8.1 常见问题排查

**问题 1：学生看不到"报名"按钮**
- 检查 `menu.js` 中学生角色的按钮配置
- 确认用户角色字段正确（应为"学生"）
- 清除浏览器缓存后刷新

**问题 2：教师审核后学生看不到审核结果**
- 检查 `jingsaibaoming` 表的 `sfsh` 字段是否更新
- 确认前端列表页是否正确渲染审核状态
- 检查浏览器控制台是否有 JavaScript 错误

**问题 3：权限不足 403 错误**
- 检查 `user_role_permission` 表中的权限配置
- 确认 `UserPermissionServiceImpl` 中的权限验证逻辑
- 查看后端日志确认请求路径
