# 登录页面访问问题修复说明

## 问题描述

访问登录页面时显示：
```json
{"msg":"请先登录","code":401}
```

而不是显示正常的登录页面。

## 问题原因

权限拦截器配置过于严格，将前端页面（包括登录页面）也拦截了，导致未登录用户无法访问登录页面。

## 已完成的修复

已修改 `InterceptorConfig.java`，放行了以下路径：

### 放行的路径

1. **前端页面资源**
   - `/admin/**` - 后台管理页面
   - `/front/**` - 前台页面
   - `/**/*.html` - HTML 文件
   - `/**/*.js` - JavaScript 文件
   - `/**/*.css` - CSS 文件
   - `/**/*.png`, `/**/*.jpg`, `/**/*.jpeg`, `/**/*.gif`, `/**/*.ico` - 图片文件
   - `/**/*.woff2` - 字体文件

2. **公开接口**
   - `/users/login` - 登录接口
   - `/users/register` - 注册接口
   - `/users/resetPass` - 重置密码
   - `/registration/**` - 统一注册接口
   - `/file/download` - 文件下载
   - `/jingsaixinxi/list` - 竞赛列表（公开）
   - `/jingsaixinxi/page` - 竞赛分页（公开）
   - `/jingsaixinxi/detail/**` - 竞赛详情（公开）

3. **其他**
   - `/static/**` - 静态资源
   - `/actuator/**` - 监控端点

## 常见问题

### Q1: 修改后仍然显示 401 错误

**原因**: 项目未重新编译，旧代码仍在运行

**解决方法**:
```bash
# 1. 停止当前运行的应用（Ctrl+C）

# 2. 清理并重新编译
mvn clean compile

# 3. 重新启动
java -jar target/springbootrd362.jar
```

### Q2: 页面空白或样式丢失

**原因**: 静态资源路径配置问题

**解决方法**:

检查 `InterceptorConfig.java` 中的资源路径配置：
```java
@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
    registry.addResourceHandler("/**")
            .addResourceLocations("classpath:/resources/")
            .addResourceLocations("classpath:/static/")
            .addResourceLocations("classpath:/admin/")
            .addResourceLocations("classpath:/front/")
            .addResourceLocations("classpath:/public/");
    super.addResourceHandlers(registry);
}
```

确认前端文件已正确构建：
```bash
# 检查 dist 目录是否存在
dir src\main\resources\admin\admin\dist
```

如果不存在，重新构建前端：
```bash
cd src/main/resources/admin/admin
npm run build
```

### Q3: 登录后仍然提示 401

**原因**: Token 未正确传递

**解决方法**:

1. **清除浏览器缓存**
   - 按 `Ctrl+Shift+Delete`
   - 清除缓存和 Cookie

2. **检查浏览器控制台**
   - 按 `F12` 打开开发者工具
   - 查看 Console 和 Network 标签
   - 检查请求是否携带 Token

3. **检查后端日志**
   - 查看是否有登录成功的日志
   - 确认 Token 是否正确生成

## 技术说明

### 拦截器配置原理

```java
@Configuration
public class InterceptorConfig extends WebMvcConfigurationSupport {
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(getAuthorizationInterceptor())
            .addPathPatterns("/**")              // 拦截所有路径
            .excludePathPatterns(                // 排除以下路径（不拦截）
                "/admin/**",                     // 前端页面
                "/users/login",                  // 登录接口
                // ... 其他排除路径
            );
    }
}
```

### 工作流程

1. **用户访问登录页面**
   - URL: `/admin/dist/index.html#/login`
   - 匹配排除规则 `/admin/**`
   - ✅ 不拦截，直接返回页面

2. **用户点击登录按钮**
   - 发送请求：`POST /users/login`
   - 匹配排除规则 `/users/login`
   - ✅ 不拦截，执行登录逻辑

3. **登录成功后访问受保护资源**
   - URL: `/jingsaixinxi/page`
   - 不匹配排除规则
   - ❌ 拦截，检查 Token
   - ✅ Token 有效，放行
