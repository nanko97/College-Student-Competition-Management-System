# 作品页面响应式快速测试指南

## 🚀 快速开始

本指南帮助你快速验证作品页面的响应式适配效果。

## 📱 方法一：浏览器开发者工具（推荐）

### Chrome/Edge 浏览器

1. **打开作品页面**
   - 学生端：访问"我的作品"页面
   - 教师端：访问"作品管理"页面

2. **打开开发者工具**
   - 按 `F12` 或 `Ctrl+Shift+I` (Windows)
   - 或右键点击页面 → "检查"

3. **切换到设备模式**
   - 点击左上角的"切换设备工具栏"图标 📱
   - 或按 `Ctrl+Shift+M`

4. **选择测试设备**

   #### 测试手机视图：
   ```
   设备选择：iPhone SE
   分辨率：375 x 667
   缩放：100%
   ```
   
   检查要点：
   - [ ] 统计卡片是否垂直堆叠显示
   - [ ] 表格是否可以横向滚动
   - [ ] 按钮大小是否适合触摸
   - [ ] 对话框宽度是否为95%
   - [ ] 文字是否清晰可读

   #### 测试平板视图：
   ```
   设备选择：iPad
   分辨率：768 x 1024
   缩放：100%
   ```
   
   检查要点：
   - [ ] 布局是否合理
   - [ ] 间距是否适当
   - [ ] 所有功能是否正常

   #### 测试自定义尺寸：
   ```
   选择：Responsive
   拖动边缘调整窗口大小
   观察布局变化
   ```

5. **旋转测试**
   - 点击旋转图标 🔄
   - 测试横屏和竖屏显示

### Firefox 浏览器

1. 按 `F12` 打开开发者工具
2. 点击"响应式设计模式"图标 (Ctrl+Shift+M)
3. 选择设备进行测试

## 🔍 常见问题排查

### 问题1：表格内容被截断

**检查**：
- 是否启用了横向滚动？
- 表格最小宽度是否设置？

**解决**：
```css
.table-content {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
```

### 问题2：按钮太小难以点击

**检查**：
- 按钮是否有足够的大小？
- 是否有足够的间距？

**解决**：
```css
@media screen and (max-width: 768px) {
  .el-button--mini {
    padding: 8px 12px;
    font-size: 13px;
  }
}
```

### 问题3：对话框超出屏幕

**检查**：
- 对话框宽度是否设置为百分比？
- 是否有最大宽度限制？

**解决**：
```css
@media screen and (max-width: 768px) {
  ::v-deep .el-dialog {
    width: 95% !important;
    margin: 5vh auto !important;
  }
}
```

### 问题4：文字重叠或溢出

**检查**：
- 是否设置了文本溢出处理？
- 字体大小是否合适？

**解决**：
```css
.el-table .cell {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

## 📊 性能测试

### 加载速度测试

1. 打开开发者工具
2. 切换到"Network"标签
3. 刷新页面
4. 检查加载时间

**目标**：
- 首屏加载 < 2秒
- 完全加载 < 5秒

### 滚动性能测试

1. 在长列表页面上快速滚动
2. 观察是否卡顿
3. 检查FPS（帧率）

**目标**：
- FPS ≥ 60
- 无明显卡顿
