## 🔧 常见问题

### Q1: 编译错误 "无法解析 getTotalCount"
**解决方案**: 已修复！使用`getTotal()`方法代替。

### Q2: 编译错误 "Page构造函数参数错误"
**解决方案**: 已修复！使用`new PageUtils(records, total, limit, page)`构造方式。

### Q3: 查询日志时返回空列表
**原因**: 
1. 还没有执行任何带`@OperationLog`注解的操作
2. 数据库表未创建

**解决方案**:
1. 确认已执行SQL脚本
2. 执行一些操作（如添加竞赛、审核报名等）
3. 再次查询

### Q4: 日志没有写入数据库
**原因**: 
1. AOP切面未生效
2. Service注入失败

**解决方案**:
1. 重启后端服务
2. 查看控制台是否有错误日志
3. 检查`OperationLogAspect.java`是否有`@Component`注解

## 💡 高级用法


## 📊 监控和维护

### 查看日志数量
```sql
SELECT COUNT(*) as total_count FROM operation_log;
SELECT DATE(create_time) as date, COUNT(*) as count 
FROM operation_log 
GROUP BY DATE(create_time) 
ORDER BY date DESC 
LIMIT 30;
```

### 查看最活跃的操作人
```sql
SELECT operator, COUNT(*) as count 
FROM operation_log 
GROUP BY operator 
ORDER BY count DESC 
LIMIT 10;
```

### 查看失败的操作
```sql
SELECT * FROM operation_log 
WHERE status = '失败' 
ORDER BY create_time DESC 
LIMIT 20;
```
