# 🚀 旧前端（Vue 2 + Element UI）手动启动指南

## ⚠️ 重要提示

**您现在看到的是新前端页面，需要按照以下步骤启动旧前端！**

---

## 📋 启动步骤（请按顺序执行）

### 第1步：启动后端服务（必须先启动）

1. **打开一个新的PowerShell窗口**
2. **复制并粘贴以下命令**：

```powershell
cd D:\BYSJ\BYSJ\Springboot_BYSJ
mvn spring-boot:run
```

3. **等待后端启动完成**（约60秒）
4. **看到以下信息表示启动成功**：
   ```
   Started SpringbootSchemaApplication in XX seconds
   ```

---

### 第2步：启动旧前端服务

1. **打开另一个新的PowerShell窗口**
2. **复制并粘贴以下命令**：

```powershell
cd D:\BYSJ\BYSJ\Springboot_BYSJ\src\main\resources\admin\admin
npm run serve
```

3. **等待前端编译完成**（约30秒）
4. **看到以下信息表示启动成功**：
   ```
   App running at:
   - Local:   http://localhost:8081/
   ```

---

### 第3步：访问旧前端

在浏览器中打开：**http://localhost:8081**

**注意**：如果页面还是显示新前端，请：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+F5）
3. 或者使用无痕模式打开

## ✅ 如何确认是旧前端？

**旧前端（Vue 2 + Element UI）的特点**：
- 登录页面有科技风格的渐变背景动画
- 使用Element UI组件（按钮、表格等样式不同）
- 页面标题显示"竞赛后台管理系统"
- 登录后左侧菜单是Element UI的样式

**新前端（Vue 3 + Naive UI）的特点**：
- 登录页面是Naive UI的风格
- 使用Naive UI组件
- 页面更现代化

## ❓ 常见问题

### Q1: 后端启动失败？
**检查**：
- MySQL数据库是否启动
- 数据库用户名密码是否正确（application.yml第41-42行）
- 9090端口是否被占用

### Q2: 前端启动失败？
**检查**：
- 是否安装了依赖：`npm install`
- 8081端口是否被占用
- Node.js版本是否正确

### Q3: 页面空白或404错误？
**原因**：后端服务未启动或API路径错误
**解决**：确保后端服务正在运行

### Q4: 还是看到新前端页面？
**解决**：
1. 确认访问的是 `http://localhost:8081`（不是8082）
2. 清除浏览器缓存
3. 使用无痕模式
4. 检查是否有其他服务占用了8081端口
