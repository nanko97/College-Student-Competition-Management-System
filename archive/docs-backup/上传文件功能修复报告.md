# 上传文件功能修复报告

## 问题描述
项目中的文件上传功能存在多个问题，导致缴费凭证等图片无法正常上传和显示。

## 根本原因分析

### 1. 后端接口缺失
- **问题**：前端调用 `jingsai/jiaofei/upload-pingzheng` 接口，但后端未实现该接口
- **影响**：学生无法上传缴费凭证

### 2. 静态资源映射配置错误
- **问题**：WebMvcConfig中使用了 `file:./target/classes/static/upload/` 绝对路径
- **影响**：IDE运行时可能找不到上传的文件，导致图片404

### 3. 前端图片路径处理不统一
- **问题**：部分页面使用自定义的 `getBaseUrl()` 方法，而非全局的 `$imageUrl` 方法
- **影响**：图片URL拼接错误，导致图片无法显示

### 修复2：修正静态资源映射配置

**文件**：`WebMvcConfig.java`

**修改前**：
```java
String uploadPath = "file:" + System.getProperty("user.dir") + "/target/classes/static/upload/";
```

**修改后**：
```java
String uploadPath = "classpath:/static/upload/";
```

**原因**：
- IDE运行时，文件上传到 `target/classes/static/upload/`
- 使用 `classpath:/static/upload/` 可以正确映射到该类路径下的资源
- 兼容jar包运行时的情况

## 技术要点说明

### 1. 全局图片URL处理方法
项目在 `utils/imageUrl.js` 中定义了全局方法：

```javascript
const BASE_URL = 'http://localhost:8080/BYSJ_Springboot'

export function imageUrl(url) {
  if (!url) return ''
  // 如果已经是完整URL，直接返回
  if (url.startsWith('http://') || url.startsWith('https://')) {
    return url
  }
  // 相对路径，拼接基础URL
  return BASE_URL + '/' + url
}
```

在 `main.js` 中注册到Vue原型：
```javascript
Vue.prototype.$imageUrl = imageUrl
```

### 2. 文件上传流程
```
学生选择图片
    ↓
前端调用 /jingsai/jiaofei/upload-pingzheng
    ↓
后端验证文件（大小、类型）
    ↓
保存到 static/upload/ 目录
    ↓
生成文件名：1234567890_photo.jpg
    ↓
数据库存储：upload/1234567890_photo.jpg
    ↓
更新缴费记录状态为"已缴费"
    ↓
前端通过 $imageUrl() 拼接完整URL
    ↓
浏览器访问：http://localhost:8080/BYSJ_Springboot/upload/1234567890_photo.jpg
    ↓
WebMvcConfig映射到 classpath:/static/upload/
    ↓
成功显示图片
```

## 注意事项

### 1. 开发环境 vs 生产环境
- **开发环境（IDE）**：文件保存到 `target/classes/static/upload/`
- **生产环境（jar包）**：文件保存到 `./upload/` 目录
- 当前配置优先适配IDE开发环境

### 2. 文件命名规则
使用时间戳作为前缀，避免文件名冲突：
```
1714234567890_原始文件名.jpg
```

### 3. 数据库字段
- `jingsai_jiaofei_jilu` 表的 `pingzheng_tupian` 字段存储相对路径
- 示例值：`upload/1714234567890_photo.jpg`

### 4. 支持的图片格式
- jpg
- png
- jpeg
