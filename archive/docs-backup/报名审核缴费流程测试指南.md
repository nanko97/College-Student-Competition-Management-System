## 🧪 测试场景

### 场景1：免费竞赛报名流程 ✅

#### 步骤1：学生报名
1. 使用学生账号登录
2. 进入"竞赛报名"页面
3. 选择一个**不需要缴费**的竞赛（shifou_shoufei = "否"）
4. 填写报名信息并提交

**预期结果**：
- ✅ 提示："报名成功，请等待教师审核"
- ✅ 报名记录的 `sfsh` 字段为 "待审核" 或 "否"
- ✅ **没有**生成缴费记录

#### 步骤2：教师审核通过
1. 使用教师账号登录
2. 进入"竞赛报名管理"页面
3. 找到刚才的报名记录
4. 点击"修改"按钮
5. 将 `sfsh` 字段改为 "是"
6. 保存

**预期结果**：
- ✅ 提示："修改成功"
- ✅ 自动生成缴费记录，状态为"已通过"
- ✅ 缴费金额为 0.00
- ✅ 审核意见："该竞赛无需缴费，系统自动通过"
- ✅ 报名表的 `ispay` 字段更新为 "已缴费"

#### 步骤3：验证学生可以提交作品
1. 切换回学生账号
2. 进入"我的作品"页面
3. 尝试提交作品

**预期结果**：
- ✅ 可以正常提交作品

## 🔍 数据库验证SQL

### 1. 检查报名记录
```sql
SELECT id, xuehao, jingsaimingcheng, sfsh, ispay, shhf 
FROM jingsaibaoming 
ORDER BY id DESC 
LIMIT 10;
```

### 2. 检查缴费记录
```sql
SELECT j.id, j.baoming_id, j.xuehao, j.jingsaimingcheng, 
       j.jiaofei_zhuangtai, j.jiaofei_jine, j.shenhe_yijian
FROM jingsai_jiaofei_jilu j
ORDER BY j.id DESC 
LIMIT 10;
```

### 3. 检查关联关系
```sql
SELECT 
    b.id AS baoming_id,
    b.xuehao,
    b.jingsaimingcheng,
    b.sfsh AS baoming_status,
    b.ispay,
    j.id AS jiaofei_id,
    j.jiaofei_zhuangtai,
    j.jiaofei_jine
FROM jingsaibaoming b
LEFT JOIN jingsai_jiaofei_jilu j ON b.id = j.baoming_id
WHERE b.sfsh IN ('是', '已通过')
ORDER BY b.id DESC;
```

### 4. 检查异常数据（应该为空）
```sql
-- 查找未审核通过但有缴费记录的报名
SELECT b.id, b.sfsh, j.id AS jiaofei_id
FROM jingsaibaoming b
INNER JOIN jingsai_jiaofei_jilu j ON b.id = j.baoming_id
WHERE b.sfsh != '是' AND b.sfsh != '已通过';
```

## 🐛 常见问题排查

### 问题1：审核通过后没有生成缴费记录

**可能原因**：
1. 竞赛不存在（jingsaiId 为空且竞赛名称匹配失败）
2. 费用配置不存在

**排查方法**：
```bash
# 查看日志
tail -f logs/info.log | grep "生成缴费记录"
tail -f logs/error.log
```

**解决方案**：
- 确保报名时正确填写了竞赛ID或竞赛名称
- 确保竞赛费用配置已设置

---

### 问题2：免费竞赛审核后ispay没有更新

**可能原因**：
- 代码逻辑错误

**排查方法**：
```sql
-- 检查免费竞赛的费用配置
SELECT * FROM jingsai_feiyong WHERE shifou_shoufei = '否';

-- 检查报名记录
SELECT id, ispay FROM jingsaibaoming WHERE id = <报名ID>;
```

**解决方案**：
- 检查日志确认是否执行了 `baoming.setIspay("已缴费")`
- 确认费用配置的 `shifou_shoufei` 字段值为 "否"

---

### 问题3：历史数据还有无效缴费记录

**可能原因**：
- 数据迁移脚本未执行

**解决方案**：
```bash
# 重新执行迁移脚本
mysql -u root -p123123 BYSJ_Springboot < src/main/resources/db/migration_baoming_shenhe.sql
```
